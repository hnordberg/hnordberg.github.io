A production-grade safety defense system that uses a cascading architecture of classifiers to screen for universal jailbreaks with minimal computational overhead, cutting costs by roughly 40x compared to running a full LLM safety check on every query.

Running a large language model to evaluate every user prompt for safety violations is economically infeasible at scale. Constitutional Classifiers++ addresses this with a cascading design analogous to Viola-Jones face detection.

Mechanism:

In the first stage, a lightweight linear probe \( C_{\text{light}} \) screens all incoming traffic. It computes a jailbreak probability for each input \( x \). If this probability exceeds a conservative threshold:

\[ C_{\text{light}}(x) > \tau \]

the query is escalated to the second stage: an expensive, context-aware LLM classifier \( C_{\text{heavy}}(x) \). Because the vast majority of benign queries are filtered out by the cheap first stage, only a small fraction of traffic reaches the expensive classifier.

The approach builds on Constitutional AI (Bai et al., 2022), turning abstract constitutional principles into highly efficient binary classifiers yielding an approximate 40x reduction in compute cost without sacrificing precision.
