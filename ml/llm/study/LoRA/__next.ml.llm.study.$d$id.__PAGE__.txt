3:T421,<p>A pretrained language model's weights encode general linguistic and world knowledge that survives largely unchanged when the model is adapted to a downstream task. <a href="/ml/wiki/topics/full-fine-tune">Full fine-tuning</a> nonetheless updates every one of those weights. For a GPT-3 175B checkpoint that is 175 billion trainable parameters per task, a 1.2 TB optimizer state under <a href="/ml/wiki/topics/adam-optimizer">Adam</a>, and a 350 GB snapshot on disk for every downstream customer. The cost scales with the base model, not with how much the model actually needs to change to handle a new task.</p><p>LoRA argues the right question is not &quot;how do we make fine-tuning cheaper&quot; but &quot;what is the intrinsic rank of the weight update a task requires.&quot; The paper's answer — almost always very small, often one — motivates a reparametrization that trains a rank-\( r \) matrix in place of the full-rank update, cuts trainable parameters by up to four orders of magnitude, and incurs zero additional latency at inference.</p>4:T488,<p>Two families of prior work addressed the same cost problem and the paper positions LoRA against both. <a href="/ml/wiki/topics/adapter-layers-houlsby">Adapter layers</a> (Houlsby et al., 2019) insert small bottleneck feed-forward modules between Transformer sub-layers and train only those. <a href="/ml/wiki/topics/prefix-tuning-and-prompt-tuning">Prefix and prompt tuning</a> (Li and Liang, 2021; Lester et al., 2021) prepend trainable continuous vectors to the input sequence while keeping the backbone frozen.</p><p>Each has a structural cost. Adapters add depth to the computation graph: every forward pass has to run through the extra bottleneck sub-layer even at deployment, and Table 1 of the paper reports a 30.3% inference-latency increase on GPT-2 Medium at batch size 1, sequence length 128, compared to the un-adapted base model. Prefix tuning consumes positions in the context window that the model can no longer use for real input, and the paper documents that performance is non-monotone in prefix length — accuracy drops as more tokens are reserved.
