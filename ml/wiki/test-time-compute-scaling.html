Test-Time Compute Scaling

A paradigm asserting that allocating vast computational resources at inference time (via search, resampling, or iterative refinement) often yields higher accuracy gains on complex logic problems than indiscriminately scaling up pre-training model parameters.

Illustration

Historically, scaling laws concentrated on training compute (Chinchilla). Test-time compute emphasizes expanding the inference budget. If an LLM is a system 1 reflex, test-time compute invokes system 2 thinking.

Primary Mechanisms

  • Best-of-N Sampling: Generate \( N \) stochastic answers and select the majority vote (self-consistency) or use a Process Reward Model. Plurality logic scales logarithmically: \( P_{\text{success}} = 1 - (1 - p)^N \).
  • Tree of Thoughts (ToT): Unfolding the reasoning path into branching nodes, utilizing lookahead heuristics and backtracking upon reaching logic failures.
  • Iterative Refinement: The LLM forcefully critiques and edits its own preliminary drafts.

For mathematically difficult logic problems, rigorous inference computation allows a 7B parameter model to radically outperform a 70B parameter model answering optimally in a single autoregressive pass. This fundamentally shifts deployment capital costs from GPU RAM footprints toward sustained inference latency.
