Sparsely-Gated Mixture of Experts (MoE)

A sparsely-gated Mixture of Experts (MoE) conditional computation architecture in which a learned routing network directs each input token dynamically to only a small subset of parallel expert sub-networks, enabling exponential parameter scaling without proportional per-example computation.

Standard dense neural networks activate every parameter for every input. Mixture of Experts (MoE) breaks this constraint: the model contains \(n\) expert networks \(E_1, \ldots, E_n\) (typically block feed-forward layers), and a gating network \(G\) selects which experts process each individual input token. For an input \(x\), the output is:

\[y = \sum_{i=1}^{n} G(x)_i \cdot E_i(x)\]

where \(G(x) \in \mathbb{R}^n\) is sparse — most entries are forced to zero structurally. Sparsity is achieved by keeping only the top-\(k\) gate values (typically \(k = 1\) or \(2\)) and zeroing the rest unconditionally via a noisy top-k gating mechanism:

\[G(x) = \text{Softmax}\big(\text{TopK}(H(x), k)\big), \quad H(x)_i = (x \cdot W_g)_i + \epsilon_i \cdot \text{Softplus}((x \cdot W_{\text{noise}})_i)\]

where \(\epsilon_i \sim \mathcal{N}(0,1)\) is tunable exploration noise added during training to encourage diverse routing distributions across experts.

Load balancing. Without auxiliary intervention, the gating network collapses into constantly routing tokens purely to its favored few initial experts. To prevent it, an auxiliary importance loss / load-balancing loss penalizes the coefficient of variation of token distributions across the cluster, enforcing an active roughly equal fractional routing behavior.

The original Sparsely-Gated MoE inserted MoE between LSTMs achieving 137B scalable models before the concept ultimately merged perfectly with Transformers resulting in systems like Switch Transformer, NLLB, and Mixtral. This was vital in enabling huge models with constant inference speed due to sparsity.
