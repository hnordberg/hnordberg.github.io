  • Reinforcement learning (RL)A machine learning paradigm where an agent learns to make sequential decisions by interacting with an environment, receiving reward signals, and optimizing a policy to maximize cumulative long-term return.
  • Reward SignalA scalar value returned by the environment (or a learned model) that indicates the desirability of an agent's recent action, serving as the sole objective function optimized in Reinforcement Learning.
  • Policy (Reinforcement Learning)A mapping from environment states to actions (or a probability distribution over actions) defining an RL agent's behavior. In modern LLMs, the neural network itself embodies the policy layer $\pi_\theta(a \mid s)$.
  • Value FunctionA mapping that evaluates how "good" it is for an agent to be in a given state (or to take a given action from that state), defined mathematically as the expected cumulative future reward under a specific policy.
  • GRPO (Group Relative Policy Optimization)A novel reinforcement learning scheme designed by DeepSeek that optimizes language models without utilizing a Critic / Value function network, heavily reducing the massive GPU memory footprint required by traditional Proximal Policy Opti…
  • REINFORCE AlgorithmThe classical policy-gradient algorithm (Williams, 1992): estimate \( \nabla_\theta J(\theta) \) by Monte-Carlo-sampling trajectories and weighting the log-probability of each action by the (possibly baseline-subtracted) return. All modern policy-gradient methods — A2C, PPO, GRPO, RLHF — are descendants of this one estimator.
  • Generalized Advantage Estimation (GAE)A family of advantage estimators that interpolates between low-variance 1-step TD residuals and high-variance Monte-Carlo returns, indexed by a parameter \( \lambda \in [0, 1] \). GAE is the default advantage signal in PPO and most modern on-policy RL.
  • Actor–Critic MethodsA hybrid policy-gradient framework that learns both a policy ("actor") and a value function ("critic") simultaneously. The critic's estimates serve as a learned baseline that sharply reduces the variance of the policy gradient, enabling stable on-policy RL at scale.
  • Process Reward Models (PRM) vs Outcome Reward Models (ORM)Two ways to score a multi-step solution: an ORM gives a single scalar for the final answer; a PRM grades each intermediate step. PRMs provide denser, per-step signal — essential for training and search in reasoning models, but expensive to annotate and vulnerable to reward hacking.
  • KTO (Kahneman–Tversky Optimization)A preference-optimization objective that trains on single-sided labels ("this completion is good" / "this completion is bad") instead of pairwise comparisons. KTO uses a reference-model-relative utility inspired by Kahneman–Tversky prospect theory: loss-aversion asymmetrically penalises completions that fall below the reference-implicit utility threshold.
  • RLAIF (RL from AI Feedback)Replace the human annotator in RLHF with a strong LLM that generates preference labels from a rubric. RLAIF scales preference collection by orders of magnitude, achieves parity with RLHF on helpfulness/harmlessness, and is the dominant pipeline for safety and reasoning alignment at modern-frontier scale.
  • Reasoning Models (o1 / R1-style Long-CoT)A class of LLMs that spend disproportionate <i>inference</i> compute on internal chain-of-thought before answering. OpenAI's o1 and DeepSeek-R1 showed that pure-RL training on verifiable reward (math, code) elicits emergent behaviours — backtracking, self-correction, hypothesis enumeration — and that test-time compute scales monotonically with accuracy.
  • Process SupervisionTraining a model (usually a process reward model) on step-level correctness labels rather than only on final-answer correctness. Process supervision gives denser credit assignment, makes verification tractable for long reasoning chains, and underpins the best published results on MATH.
  • Monte Carlo Tree Search for LLM ReasoningApply the AlphaZero-style MCTS framework to tree-structured LLM reasoning: expand candidate continuations as children of the current partial solution, evaluate leaves with a value/PRM, and select promising branches via UCB. MCTS is the default search procedure for deliberate test-time reasoning when a step-level scorer is available.
